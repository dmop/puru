README and all public-facing docs should follow best practices from high-conversion, production-grade open-source libraries.

**Why:** The user wants puru to attract attention from experienced software engineers. The README is the primary conversion surface — it needs to be polished, credible, and concise like top-tier OSS projects (e.g., Zod, tRPC, Hono, Drizzle).

**How to apply:** When writing or editing README, docs, or any public-facing text: lead with a clear value prop, show before/after comparisons, keep examples minimal and copy-pasteable, highlight zero-dep / lightweight nature, use badges, include benchmark data, and avoid filler. Every sentence should earn its place.
